# Excel Agent

Локальный аналог AutoGPT для Excel-отчётов. Агент читает `.xlsx`, генерирует Python-код через локальную LLM, выполняет его в песочнице и при ошибке пробует снова.

## Возможности

- Запрос на русском: «Возьми data.xlsx, посчитай прибыль, выдели отрицательные красным, построй диаграмму»
- Автоанализ структуры Excel (листы, столбцы, типы, примеры строк)
- LangGraph-цикл: **анализ → генерация → выполнение → оценка → повтор**
- Безопасный запуск кода в подпроцессе с таймаутом и проверкой опасных импортов

## Подготовка

1. Создайте папку `models/`
2. Скачайте `.gguf` модель (рекомендуется Qwen2.5-Coder 7B или аналог)
3. Запустите `setup.bat`

## Запуск

```bat
run.bat "Возьми data.xlsx, столбцы Выручка и Расходы, посчитай прибыль" --input data.xlsx --output итог.xlsx
```

Или с явными путями:

```bat
python main.py "посчитай прибыль и построй столбчатую диаграмму" -i data.xlsx -o result.xlsx
```

Без аргументов запускается демо-задача на `data.xlsx`.

## Архитектура

```
START → analyze_excel → prepare_prompt → generate_code → execute_code → evaluate
                              ↑                                              |
                              └──────────────── retry ───────────────────────┘
```

| Файл | Назначение |
|------|------------|
| `main.py` | LangGraph-агент и CLI |
| `excel_analyzer.py` | Анализ структуры Excel для контекста LLM |
| `code_executor.py` | Песочница: AST-проверка + подпроцесс с таймаутом |
| `excel_helpers.py` | Готовые функции: формулы, условное форматирование, BarChart |
| `output_validator.py` | Проверка результата: есть ли диаграмма, формулы, не red/black |
| `prompt_storage.py` | Системные промпты |

## Зависимости

- llama-cpp-python — локальная LLM
- langgraph — граф состояний с циклом повторов
- pandas, openpyxl, matplotlib — работа с Excel и графиками
