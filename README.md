# Excel Agent

Локальный аналог AutoGPT для Excel-отчётов. Агент читает `.xlsx`, генерирует Python-код через локальную LLM, выполняет его в песочнице и при ошибке пробует снова.

## Возможности

- Запрос на русском: «Возьми data.xlsx, посчитай прибыль, выдели отрицательные красным, построй диаграмму»
- Автоанализ структуры Excel (листы, столбцы, типы, примеры строк)
- 7-этапный LangGraph-цикл с валидацией кода и результата
- Безопасный запуск кода в подпроцессе с таймаутом и проверкой опасных импортов
- Все взаимодействие с Excel через openpyxl (формулы, диаграммы, условное форматирование)

## Подготовка

1. Создайте папку `models/`
2. Скачайте `.gguf` модель (рекомендуется Qwen2.5-Coder 7B или аналог)
3. Запустите `setup.bat`

## Запуск

```bat
run.bat "Возьми data.xlsx, столбцы Выручка и Расходы, посчитай прибыль" --input data.xlsx --output итог.xlsx
```

Или с явными путями:

```bat
python main.py "посчитай прибыль и построй столбчатую диаграмму" -i data.xlsx -o result.xlsx
```

Без аргументов запускается демо-задача на `data.xlsx`.

## Архитектура

```
START → analyze_excel → prepare_prompt → prepare_code_generation_prompt → generate_code → generated_code_validation → execute_code → evaluate
                                    ↑                    ↑                       |                   |
                                    │                    └─── fix prompt ────────┘                   │
                                    │                                                                │
                                    └─────────────────────────── retry ──────────────────────────────┘
```

### Этапы графа

| # | Этап | Назначение |
|---|------|------------|
| 1 | `analyze_excel` | Анализ структуры Excel для контекста LLM |
| 2 | `prepare_prompt` | Уточнение и детализация пожеланий пользователя |
| 3 | `prepare_code_generation_prompt` | Сопоставление данных таблиц с запросом, создание жёстких рамок для генерации кода |
| 4 | `generate_code` | Генерация Python-кода через LLM |
| 5 | `generated_code_validation` | Проверка кода на соответствие требованиям (макс. 1 повтор) |
| 6 | `execute_code` | Выполнение кода в песочнице |
| 7 | `evaluate` | Проверка выходного Excel-файла через LLM |

### Валидация

- **Код**: проверяется перед выполнением. При неудаче — отправляется на доработку с промптом от валидатора (макс. 1 раз).
- **Результат**: проверяется после выполнения. При неудаче — полный retry с этапа `prepare_prompt`.

## Файлы

| Файл | Назначение |
|------|------------|
| `main.py` | LangGraph-агент и CLI |
| `excel_analyzer.py` | Анализ структуры Excel для контекста LLM |
| `code_executor.py` | Песочница: AST-проверка + подпроцесс с таймаутом |
| `output_validator.py` | Валидация кода и результата через LLM |
| `prompt_storage.py` | Хранилище системных промптов |

## Зависимости

- llama-cpp-python — локальная LLM
- langgraph — граф состояний с циклом повторов
- openpyxl — работа с Excel (формулы, диаграммы, форматирование)
